# Item-List Field UI — アコーディオン式リデザイン

**日付:** 2026-04-14  
**対象ファイル:** `src/features/recording/DynamicForm.tsx`

---

## 問題

現在のアイテムリストフィールドは「ピル群（上部）→ 入力カード群（下部）」という2エリア分離構造になっている。項目を選ぶたびに入力カードが下に積み重なり、2〜3個選択するとスクロールが必要になる。さらに次の項目を選ぶためには上のピル群まで戻らなければならず、上下の行き来が発生する。

---

## 解決策：アコーディオン式レイアウト

各項目をボタンとして縦に並べ、ボタンの直下に入力欄を展開する。ピルエリアと入力エリアを分離しない。

---

## UI 仕様

### 項目の状態（3種類）

| 状態 | 表示 |
|------|------|
| **未選択** | グレーのフルワイドボタン（項目名のみ） |
| **選択中・展開** | ピンクのボタン（▲アイコン）＋直下に入力行 |
| **選択中・折りたたみ** | ピンクのボタン内にサマリー文字列（例: `80kg × 8回 = 640`）＋ ▼アイコン |

### 展開時の入力行レイアウト

ボタン直下に1行で表示：

```
[ 重量 (kg) ]  [ 回数 ]  640
 <input>        <input>   <合計値>
```

- サブフィールドは横並び（`grid-template-columns: 1fr 1fr auto`）
- `computedTotal` が有効な場合のみ合計値を表示
- サブフィールドが1つの場合は合計列なし

### タップ操作

- **未選択ボタンをタップ** → 選択状態になり入力欄を展開
- **展開中ボタンをタップ** → 折りたたみ（値は保持）
- **折りたたみ中ボタンをタップ** → 再展開
- **折りたたみ中ボタンを長押し（500ms）** → 選択解除（値を削除）

> 長押し削除にする理由：誤タップで値が消えるのを防ぐ。折りたたみボタンへの通常タップが「展開」なので、削除は別ジェスチャーが必要。

### サマリー文字列の生成ルール

- サブフィールドが2つ ＋ `computedTotal: true` → `{val0}{sf0.unit} × {val1}{sf1.unit} = {total}`
  - 例: `60kg × 10回 = 600`
- サブフィールドが2つ ＋ `computedTotal: false` → `{sf0.label}: {val0} / {sf1.label}: {val1}`
  - 例: `重量: 60 / 回数: 10`
- サブフィールドが1つ → `{sf0.label}: {val0}`
- 値が1つも入力されていない場合 → サマリーなし（ボタン名のみ表示）

## 実装スコープ

**変更するファイル:** `src/features/recording/DynamicForm.tsx` の `item-list` レンダリング部分のみ

**変更しないもの:**
- `expandedItems` の state 管理ロジック（再利用）
- `updateEntry` / `toggleItem` の内部ロジック（再利用、ただし toggleItem の展開/折りたたみ挙動を調整）
- 他のフィールドタイプ（number, text, select など）

**新規追加:**
- 長押し検出ロジック（`onPointerDown` + `setTimeout` + `onPointerUp`/`onPointerLeave` でキャンセル）
- サマリー文字列生成ヘルパー関数（コンポーネント内）

---

## データ構造

変更なし。`values[field.key]` は引き続き `Array<{ name: string; [subKey]: number; total?: number }>` 形式。

---

## 非機能要件

- スマホ（375px幅）で2サブフィールド＋合計が1行に収まること
- 長押しタイムアウトは 500ms
- アニメーションは不要（シンプルな表示/非表示切り替え）
